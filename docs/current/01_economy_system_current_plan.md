# NeksMarket 经济系统新方案：真实与游戏平衡版

更新时间：2026-05-08

## 1. 为什么需要新方案

原计划的核心目标是：

- 建立价格档案。
- 用锚点价和配方图推导物品价格。
- 用库存和玩家市场成交价做动态调价。
- 让系统商店支持货架和多种定价模式。

这些方向仍然正确，但现在我们明确了一个更重要的问题：

> Minecraft 的资源可以被无限生成，不能直接照搬真实世界经济。

真实经济依赖有限资源、大量市场参与者、政策调控和复杂生产链。MC 服务器只有有限玩家、单机或单服算力，以及大量自动化农场。如果系统按固定价长期收购可再生资源，刷铁机、刷怪塔、作物农场会把货币变成无限资源。

因此，新方案不是推翻原计划，而是在原计划上加一个更高层的经济约束：

> 自动定价负责“物品大概值多少钱”，动态经济负责“无限资源不能永久稳定印钞”。

## 2. 新核心原则

### 2.1 自动定价不等于自动兑换

系统能算出价格，不代表系统一定要收购或出售。

必须坚持：

- 有参考价，不等于系统可买卖。
- 可回收，不等于可出售。
- 可出售，必须由管理员货架控制。
- 高阶资源不能被低阶无限资源稳定换出。

### 2.2 系统商店不是万能商店

系统商店定位为服务器经济缓冲器：

- 回收常见资源，给玩家基础收入。
- 投放基础资源，避免基础物资短缺。
- 通过价格衰减抑制无限刷钱。
- 高阶、终局、模组关键物品主要交给玩家交易行。

### 2.3 自动化可以赚钱，但不能永久印钞

自动化是 MC 的乐趣，不应该被完全否定。

合理目标是：

- 农场刚建成时收益明显。
- 大量产出后价格逐步衰减。
- 玩家需要寻找新的市场缺口。
- 不存在一个永远稳定最高收益的刷钱机器。

## 3. 经济系统分层

### 3.1 初始基础价层

用途：

> 判断一个物品在“没有市场波动时”大概值多少钱。

组成：

- 少量人工锚点价。
- 自然资源规则。
- 掉落物规则。
- 稀有度规则。
- 配方推导。
- 玩家市场成交价参考。

基础价不直接等于最终交易价。

### 3.2 物品分级层

每个物品应该有经济分级。

建议分级：

- `COMMON_RENEWABLE`：常见可再生物，如作物、圆石、木头。
- `COMMON_NON_RENEWABLE`：普通不可再生物，如部分矿物。
- `INDUSTRIAL_RENEWABLE`：高度自动化产物，如铁锭、骨粉、刷怪掉落物。
- `RARE_RESOURCE`：稀有资源，如钻石、远古残骸。
- `BOSS_DROP`：Boss 或特殊掉落物。
- `PROGRESSION_LOCKED`：推进关键物品，如高级机器、终局材料。
- `PLAYER_MARKET_ONLY`：只允许玩家市场。

分级决定：

- 是否允许系统回收。
- 是否允许系统出售。
- 回收价衰减速度。
- 是否限购或禁止自动上架。

### 3.3 系统回收层

用途：

> 玩家把物品卖给系统时，决定系统给多少钱。

推荐采用 `Final.md` 中的三层压力模型，而不是单纯库存压力或单纯近期压力。

核心公式：

```text
buyPrice = baseBuyPrice × exp(-alpha × S^gamma - beta × C - delta × Δ)
```

含义：

- `baseBuyPrice`：基础回收价。
- `S`：累计净流入总量，永久保留，用于长期价格中枢下移。
- `C`：近期/中期流入压力的 EMA，用于持续产线的市场记忆。
- `Δ`：本次交易数量，用于单次大量倾倒的短期冲击。
- `alpha`：长期压制强度。
- `gamma`：长期压制曲线形状，建议默认 `0.3`。
- `beta`：中期记忆压制强度。
- `delta`：单次交易滑点强度。

效果：

- 少量出售时价格接近基础价。
- 一次性大量倾倒时，本次成交价格明显下滑。
- 持续自动化产线会让中期压力维持高位。
- 长期大量流入会让价格中枢永久下移。
- 停止供给后，中期压力会恢复，但长期中枢不会完全回到原点。

状态更新：

```text
S_new = S_old + Δ
C_new = exp(-lambda × elapsedTicks) × C_old + lambda × Δ
```

任意时刻查询展示价格：

```text
C_now = exp(-lambda × elapsedTicks) × C_last
displayBuyPrice = baseBuyPrice × exp(-alpha × S^gamma - beta × C_now)
```

说明：

- 查询价格不包含 `delta × Δ`，因为没有真实交易数量。
- 执行交易时才把 `delta × Δ` 作为滑点加入。
- 该方案可以惰性求值，每个物品只保存 `S`、`C_last`、`lastTick`，查询和交易都是 `O(1)`。

不同物品分级使用不同参数：

```text
手采资源：k 小，衰减慢。
普通可再生物：k 中等。
工业可再生物：k 大，衰减快。
稀有掉落物：k 小，但可能有每日限量。
推进关键物品：默认不回收或只玩家市场。
```

后续实现时，上面这组 `k` 应细化为不同分级的 `alpha / beta / delta / lambda / minRatio`。

### 3.4 系统出售层

用途：

> 玩家从系统买物品时，决定售价。

系统出售应该以货架和库存为核心。

基础公式：

```text
sellPrice = baseSellPrice × stockFactor × marketFactor
```

库存系数：

```text
stockFactor = clamp((targetStock / max(stock, 1)) ^ elasticity, minRatio, maxRatio)
```

含义：

- 库存低，售价上涨。
- 库存高，售价下降或保持。
- 不能无限涨，也不能无限跌。

注意：

系统出售只对管理员上架的货架商品生效。系统不能因为能算价就自动出售所有物品。

### 3.5 玩家交易行层

玩家交易行承担真实市场部分。

用途：

- 高阶物品定价。
- 稀有物品流通。
- 模组推进物品交易。
- 玩家之间真实供需。

玩家成交价只温和影响参考价，不能完全控制系统价格。

建议公式：

```text
marketFactor = 1 + (marketVWAP / basePrice - 1) × marketWeight × confidence
```

限制：

- 成交太少时置信度低。
- 单个玩家刷成交不能强行操纵系统价。
- 市场因子要有上下限。

### 3.6 货币回收层

如果系统只发钱不收钱，长期一定通胀。

必须保留货币出口：

- 系统售价高于系统回收价。
- 玩家交易税。
- 上架手续费。
- 传送、修理、改名、租地等服务器服务费。
- 高阶功能解锁费。

最低原则：

```text
systemBuyPrice < systemSellPrice
```

中间价差就是最基础的货币回收机制。

## 4. 与原计划的关系

### 4.1 保留原计划内容

这些方向继续保留：

- 价格档案系统。
- 锚点价。
- 玩家成交记录。
- 玩家市场 VWAP。
- 系统商店货架制。
- PriceMode。
- 配方图推导。
- 自然资源和掉落物规则。
- 出货箱批量回收。
- tooltip 显示回收价。
- 后台 warmup。

### 4.2 需要调整的内容

原计划中“动态库存调价”需要升级为：

```text
库存调价 + S/C/Δ 回收压力衰减 + 物品分级 + 货币回收
```

原计划中“系统收购页可以自动生成”也要更谨慎：

```text
可以自动判断可回收，但必须受物品分级和衰减规则控制。
```

系统购买页继续严格货架制。

## 5. 新开发阶段

### 阶段 1：经济分级模型

目标：

- 给物品引入经济分级。
- 默认按规则自动分类。
- 配置允许覆盖。

建议新增：

```text
EconomicTier
EconomicPolicy
EconomicPolicyRegistry
```

验收：

- `/market price item <物品>` 能显示经济分级。
- 常见物品能被合理归类。

### 阶段 2：回收压力曲线

目标：

- 将系统回收价从简单库存压力升级为 `Final.md` 的 S/C/Δ 指数衰减模型。
- 使用惰性求值，避免每 tick 扫描所有物品。

建议配置：

```text
buyPressureGamma
buyPressureLongAlpha
buyPressureMemoryBeta
buyPressureTradeDelta
buyPressureMemoryLambda
buyPressureMinRatio
```

验收：

- 少量卖铁价格接近基础价。
- 一次性大量卖铁，本次成交出现滑点。
- 连续大量卖铁后回收价明显下降。
- 等待一段时间后价格部分恢复，但长期中枢仍保留痕迹。

### 阶段 3：系统出售库存曲线

目标：

- 系统出售继续走货架。
- 售价根据库存目标变化。
- 禁止自动出售未上架物品。

验收：

- 库存低时售价上涨。
- 库存恢复后售价回落。
- 未上架物品无法从系统购买。

### 阶段 4：高阶资源保护

目标：

- 避免低阶无限资源稳定兑换高阶资源。

措施：

- 高阶物品默认 `PLAYER_MARKET_ONLY` 或 `REFERENCE_ONLY`。
- Boss 掉落物默认不系统出售。
- 终局材料默认不系统回收或强限制。
- 管理员可手动固定价和限量上架。

验收：

- 钻石、远古残骸、下界之星等不会被自动万能兑换破坏。

### 阶段 5：价格解释和调试

目标：

- 管理员能看懂价格为什么这样变化。

建议命令：

```mcfunction
/market price item <物品>
/market price policy <物品>
```

输出：

- 基础价来源。
- 经济分级。
- 当前回收压力。
- 当前库存系数。
- 市场成交因子。
- 最终回收价和售价。

### 阶段 6：长期平衡工具

目标：

- 给服主调经济，而不是让服主改代码。

建议功能：

- 导出价格报告。
- 查看最高系统支出物品。
- 查看近期被大量回收的物品。
- 查看货币总发行量。
- 查看系统净流入/流出。

## 6. 推荐默认策略

### 6.1 常见可再生物

例子：

- 小麦。
- 胡萝卜。
- 原木。
- 圆石。

策略：

- 允许回收。
- 回收价低。
- 衰减中等。
- 可作为基础收入来源。

### 6.2 工业可再生物

例子：

- 铁锭。
- 骨头。
- 火药。
- 腐肉。

策略：

- 允许回收。
- 衰减快。
- 大量倾倒后收益明显下降。

### 6.3 稀有资源

例子：

- 钻石。
- 远古残骸。
- 绿宝石。

策略：

- 可显示参考价。
- 是否回收由配置决定。
- 系统出售默认需要管理员货架。
- 衰减慢但售价高。

### 6.4 Boss 和推进物品

例子：

- 下界之星。
- 龙蛋。
- 鞘翅。
- 模组终局材料。

策略：

- 默认玩家市场。
- 默认不系统回收。
- 默认不系统出售。
- 只允许管理员特殊配置。

## 7. 测试方向

### 7.1 刷铁机压力测试

目标：

验证无限铁不能永久印钞。

流程：

```mcfunction
/market system quote buy_from_player minecraft:iron_ingot 64
```

连续出售多组铁锭后，观察回收价是否下降。

### 7.2 普通玩家友好测试

目标：

验证少量出售不会被惩罚得太狠。

测试：

- 卖少量原木。
- 卖少量矿物。
- 卖少量农作物。

预期：

- 价格接近基础回收价。

### 7.3 高阶兑换测试

目标：

验证低阶资源不能稳定换高阶资源。

测试：

- 大量卖铁。
- 尝试从系统购买钻石、远古残骸、下界之星。

预期：

- 未上架的高阶资源无法购买。
- 已上架的高阶资源也应很贵、限量或受库存约束。

## 8. 当前结论

原计划没有废弃，但已经不够完整。

新的总方向应该是：

```text
价格推导系统 + 经济分级系统 + 回收压力衰减 + 货架制投放 + 玩家交易行 + 货币回收
```

一句话目标：

> 让普通玩家有稳定收入，让自动化玩家有阶段性收益，但不允许无限资源长期稳定地兑换全服高阶资源。
