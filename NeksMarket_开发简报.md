# NeksMarket 开发简报

更新时间：2026-05-08

## 项目简介

NeksMarket 是一个 Minecraft 1.21.1 NeoForge 市场模组，目前核心方向是把普通玩家市场和系统商店做成一个带动态经济逻辑的交易系统。

当前已经不只是“玩家挂单买卖”的基础市场，而是在逐步加入：

```text
动态价格
系统商店货架
系统库存
玩家成交历史
管理员可配置商品
未来的供需影响与反套利机制
```

目标不是做一个固定价格商店，而是做一个会根据市场、库存和配置规则变化的经济系统。

## 目前已经实现的功能

### 1. 玩家市场

已实现基础玩家自由市场：

```text
玩家可以出售手中物品
其他玩家可以购买挂单
卖家可以领取收益
玩家可以取消自己的挂单
挂单有过期时间
使用虚拟货币“村民币”
```

### 2. 系统商店

已实现系统商店的基础买卖：

```text
系统可以向玩家出售物品
系统可以从玩家手中收购物品
系统商店有 Buy / Sell 两个方向
Buy 页支持货架分类
管理员可以添加或删除系统商品
系统商品配置支持 reload
```

目前系统商店支持两类商品：

```text
sell_to_player：系统卖给玩家，显示在 Buy 页
buy_from_player：系统从玩家收购，显示在 Sell 页
```

### 3. 系统商店货架

系统卖给玩家的商品现在可以放入不同货架分类。

当前支持：

```text
多个货架分类
分类重命名
分类重置
系统出售商品按分类显示
如果添加系统出售商品时不填分类，会自动进入第一个货架
```

### 4. 动态价格框架

已经搭建了动态价格核心结构。

目前支持的价格模式：

```text
FIXED：固定价格
AUTO：自动价格
ANCHOR：锚定基础价格
MULTIPLIER：按基础价格倍率计算
BAND：价格区间限制
```

旧配置仍然兼容：

```text
id|type|item|unitPrice|category
```

新配置格式：

```text
id|type|item|category|priceMode|basePrice|multiplier|minPrice|maxPrice|flags
```

### 5. 成交历史与价格查询

系统已经会记录玩家市场成交历史，并提供价格查询命令。

可用命令包括：

```text
/market price hand
/market price item <item>
/market price history hand
/market price history item <item>
/market price reload
```

这些命令可以用来查看物品价格、历史成交情况，并辅助调试动态价格。

### 6. 命令补全

所有主要涉及物品名的命令，已经改成类似 `/give` 的物品参数补全。

例如输入：

```text
/market price item minecraft:iron...
```

会出现 Minecraft 原生物品 ID 补全。

### 7. 系统库存雏形

已经加入系统库存记录。

当前会记录：

```text
系统从玩家收购了多少该物品
系统向玩家卖出了多少该物品
系统当前实际库存
```

可查询：

```text
/market system stock <item>
```

当前库存已经会影响部分动态收购价：

```text
系统库存越多，非 FIXED 模式下系统继续收购该物品的价格越低
```

### 8. GUI 体验修复

已修复一些测试中发现的问题：

```text
同一个物品在系统商店出现多个价格
打开系统商店时价格首次显示为 0
reload 后新商品不立刻显示
GUI 中交易后背包不实时刷新
系统出售商品不填分类时添加后不可见
```

目前在 GUI 中买卖后，玩家快捷栏和背包数量可以实时变化。

## 目前还没完全做完的部分

### 1. 系统库存还不是完整库存系统

现在系统库存已经会记录，也会影响收购价格，但还没有完整限制系统出售。

计划中希望实现：

```text
系统库存为 0 时，非无限库存商品不能继续卖给玩家
玩家卖给系统后，系统才有库存可以卖
部分商品可以设置为 infinite_stock，不受库存限制
GUI 显示库存数量或无限库存状态
```

### 2. 高级动态商品添加命令还缺少

目前 `/market system add` 主要用于添加固定价格商品。

计划新增类似：

```text
/market system addpriced <id> <type> <item> <category> <mode> <basePrice> <multiplier> <minPrice> <maxPrice> [flags]
```

用途：

```text
管理员可以直接用命令添加 AUTO / ANCHOR / MULTIPLIER / BAND 商品
不需要手动改 config
```

### 3. 价格调试命令还可以增强

计划增加更清楚的报价调试命令，例如：

```text
/market system quote <type> <item> [count]
```

目标：

```text
不用打开 GUI，也能看到系统当前买入价、卖出价、库存影响和价格来源
```

### 4. 配置分类还需要整理

目前系统商店货架分类支持中文，但在某些控制台或配置显示里可能出现编码乱码。

未来更稳的方案：

```text
内部使用稳定 category id，例如 #1 / #2 / #3
GUI 显示名单独配置中文
避免把中文显示名直接当作逻辑 key
```

### 5. 动态经济算法还可以继续扩展

未来可以接入更多影响价格的因素：

```text
玩家市场成交均价
近期成交量
系统库存
基础物品稀有度
合成链成本
世界生成稀有度
掉落来源
反套利规则
价格上下限保护
```

## 近期开发计划

建议接下来的开发顺序：

### 阶段一：完善系统库存

优先级最高。

要做：

```text
系统出售商品检查库存
库存不足时禁止购买
支持 infinite_stock 标记
GUI 显示库存状态
交易后库存实时刷新
```

### 阶段二：管理员高级商品命令

要做：

```text
添加 addpriced 命令
支持所有 priceMode
支持 flags
继续保留物品名自动补全
命令成功后立即刷新系统商店
```

### 阶段三：调试与可视化

要做：

```text
增加系统报价查询命令
显示价格来源
显示库存影响
显示基础价、倍率、上下限
让管理员更容易判断价格为什么是这个数
```

### 阶段四：经济算法扩展

要做：

```text
让玩家市场成交历史真正影响系统价格
增加价格平滑机制
增加异常价格保护
加入更多物品基础价格数据
逐步扩展到更多商品类型
```

## 对外讨论时可以重点问的问题

如果要找其他开发者讨论，建议重点讨论这些设计问题：

```text
系统商店是否应该默认拥有无限库存？
哪些商品应该 infinite_stock，哪些商品必须依赖玩家供给？
系统收购价应该受库存影响多强？
系统售价是否也应该随库存变动？
动态价格应该偏向稳定，还是允许明显波动？
是否允许玩家通过低买高卖套利？如果不允许，怎么限制？
分类系统应该用中文名做 key，还是用固定 id + 显示名？
GUI 中库存、价格来源、动态变化要显示到什么程度？
```

## 当前项目状态一句话总结

NeksMarket 目前已经完成了从“基础玩家市场”到“动态系统商店”的第一轮核心改造：玩家交易、系统买卖、动态报价、货架分类、价格查询和库存记录都已经跑通；下一步重点是把系统库存做成真正影响系统出售的完整机制，并补齐高级管理员命令和价格调试工具。
